Create `src/simulation/spatialHash.ts` exporting a `SpatialHashGrid` class using pre-allocated Int32Array storage (zero per-frame allocations):

1. **Constructor** accepts `width`, `height`, `cellSize`, `maxEntries`:
   - `cellSize`: Use `2 * maxRadius` as cell size (ensures any circle touches at most 4 cells)
   - Compute `cols = Math.ceil(width / cellSize)` and `rows = Math.ceil(height / cellSize)`
   - Pre-allocate `cells: Int32Array(totalCells * maxPerCell)` filled with -1 (stores bubble indices per cell)
   - Pre-allocate `cellCounts: Int32Array(totalCells)` (count of items per cell)
   - `maxPerCell = 16` (generous for 400 bubbles in reasonable distribution)

2. **clear():** Reset `cellCounts.fill(0)` only (no need to clear cells array -- count controls reads)

3. **insert(index, x, y, radius):** A circle can span multiple cells. Compute min/max col/row from `(x-radius)/(cellSize)` to `(x+radius)/(cellSize)`, clamped to grid bounds. For each overlapping cell, store the bubble index if count < maxPerCell.

4. **queryNeighbors(index, x, y, radius, callback):** For the cell range covering the bubble's extent, iterate stored indices. Use `other > index` guard to check each pair only once. Call `callback(other)` for each candidate.

5. **resize(width, height):** Update cols/rows, re-allocate cells and cellCounts arrays if new total cells exceeds current capacity. This handles canvas resize.

Key constraints:
- Zero allocations in clear/insert/queryNeighbors (all pre-allocated Int32Arrays)
- All array access via flat index arithmetic (no nested arrays, no objects)
- Grid is rebuilt every frame via clear() + insert() (not incremental) -- this is intentional and fast for 400 items

Create `src/simulation/physics.ts` -- a pure-function physics module with NO React dependencies. All functions operate directly on SimulationBuffers typed arrays.

Import `SimulationBuffers` from `./state` and `SpatialHashGrid` from `./spatialHash`. Import `createNoise2D` from `simplex-noise`.

**1. Physics constants (exported for easy tuning):**
```typescript
export const PHYSICS = {
  NOISE_FREQUENCY: 0.003,       // Lower = smoother, larger flow fields
  NOISE_AMPLITUDE: 0.15,        // Force magnitude -- gentle
  NOISE_TIME_SCALE: 0.0004,     // How fast noise field evolves
  DAMPING: 0.98,                // Velocity damping per step
  MAX_VELOCITY: 1.5,            // Cap to prevent runaways
  COLLISION_ITERATIONS: 4,      // Multi-pass collision (PHYS-03)
  OVERLAP_SLOP: 0.5,            // Allow 0.5px overlap (soft-body feel)
  PUSH_STRENGTH: 0.4,           // Partial resolution per pass (soft feel)
  BOUNDARY_PADDING: 15,         // px from canvas edge
  BOUNDARY_STIFFNESS: 0.1,      // Push-back force at edge
  BOUNDARY_DAMPING: 0.8,        // Velocity damping at boundary
} as const;
```

**2. createPhysicsState(maxBubbles):** Factory function returning a physics state object containing:
   - A `SpatialHashGrid` instance (initial size, resized later)
   - The `noise2D` function from `createNoise2D()`
   - Canvas dimensions (mutable, updated on resize)

**3. applyForces(buffers, count, time, dt, noise2D):**
   - For each bubble: sample Simplex noise at `(x * NOISE_FREQUENCY, time * NOISE_TIME_SCALE)` for X force, offset +100 for Y force
   - Scale force inversely with `1 / Math.sqrt(mass + 1)` -- smaller bubbles drift faster per user decision
   - Semi-implicit Euler: update velocity first (`vx = (vx + forceX * dt) * DAMPING`), then position (`x += vx * dt`) -- PHYS-01
   - Clamp velocity to MAX_VELOCITY to prevent runaways

**4. resolveCollisions(buffers, count, grid):**
   - Run COLLISION_ITERATIONS passes (PHYS-03: 4 passes)
   - Each pass: grid.clear(), insert all bubbles, then for each bubble query neighbors
   - For each colliding pair (distSq < minDist^2 and distSq > 0.0001):
     - Compute overlap. If overlap > OVERLAP_SLOP (soft-body: allow slight overlap):
       - Mass-weighted push-back: `pushI = (1/massI) / totalInvMass`, `pushJ = (1/massJ) / totalInvMass`
       - Position correction: `correction = (overlap - OVERLAP_SLOP) * PUSH_STRENGTH`
       - Push positions apart along collision normal
       - Light velocity transfer: `relVelDot * 0.3` impulse (soft momentum, not sharp bounce)
   - Use squared distance comparison first, only compute sqrt when overlap confirmed (per research anti-patterns)

**5. enforceBoundary(buffers, count, canvasWidth, canvasHeight):**
   - For each bubble: compute min/max allowed positions as `radius + BOUNDARY_PADDING` from edges
   - If outside: clamp position, reverse and damp velocity (`*= -BOUNDARY_DAMPING`)
   - Soft bounce feel per user decision (decelerate, no jerk) -- PHYS-06

**6. initialPlacement(buffers, count, width, height):**
   - Sort indices by mass descending (largest first)
   - Largest bubbles get tight spawn radius around center (t=0 -> 0.2 * halfDiagonal)
   - Smallest bubbles get full canvas spread (t=1 -> 0.9 * halfDiagonal)
   - Random angle and distance within spawn radius for each bubble
   - NO center gravity force anywhere -- large bubbles near center is ONLY via this placement (PHYS-05 user override)

**7. stepPhysics(physicsState, buffers, count, dt, time):** Orchestrator function called once per fixed timestep:
   1. `applyForces(buffers, count, time, dt, physicsState.noise2D)`
   2. `resolveCollisions(buffers, count, physicsState.grid)`
   3. `enforceBoundary(buffers, count, physicsState.canvasWidth, physicsState.canvasHeight)`

**8. handleResize(physicsState, buffers, count, newWidth, newHeight):** Called on canvas resize:
   - Update physicsState canvas dimensions
   - Resize spatial hash grid
   - Clamp all bubble positions to new bounds (push inward if window shrank -- per user decision)

CRITICAL constraints:
- NO center gravity force. PHYS-05 is explicitly overridden by user. Do NOT add any force pulling bubbles toward center.
- NO React imports. This is a pure computation module.
- Zero allocations in hot path (stepPhysics and everything it calls). The `indices` array in `initialPlacement` is fine since it runs once.
- All typed array access by index, no destructuring in loops.
