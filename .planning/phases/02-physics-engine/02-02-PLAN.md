**Part A: Create `src/simulation/gameLoop.ts`**

Export a `createGameLoop` factory function that implements the Gaffer on Games fixed-timestep accumulator pattern (PHYS-04):

```typescript
export function createGameLoop(
  update: (dt: number, time: number) => void,
  render: () => void,
): { start: () => void; stop: () => void }
```

Implementation:
- `FIXED_DT = 1000 / 60` (16.667ms physics step)
- `MAX_FRAME_TIME = 100` (cap at 100ms to prevent spiral of death after tab backgrounding)
- `MAX_STEPS_PER_FRAME = 5` (safety cap -- prevents freeze after long tab background)
- Uses `requestAnimationFrame` for the render loop
- Accumulator pattern: on each frame, add elapsed time to accumulator, run physics steps while accumulator >= FIXED_DT, then render once
- `start()`: begins the loop (no-op if already running)
- `stop()`: cancels the requestAnimationFrame (called on component unmount)
- Reset `lastTime = 0` on start so first frame has zero delta (prevents huge initial jump)
- No React dependencies -- pure JS module

**Part B: Rewrite `src/components/BubbleCanvas.tsx`**

Replace the entire BubbleCanvas component. Remove the `drawTestBubble` and `renderCanvas` helper functions. The new component:

1. **Initialization (useEffect on mount, runs once):**
   - Import `MOCK_STOCKS` from `../data/mockStocks`
   - Import `createSimulationBuffers`, `initBuffersFromStocks` from `../simulation/state`
   - Import `createPhysicsState`, `initialPlacement`, `stepPhysics`, `handleResize` from `../simulation/physics`
   - Import `createGameLoop` from `../simulation/gameLoop`
   - Create buffers: `createSimulationBuffers(MOCK_STOCKS.length)`
   - Initialize buffers: `initBuffersFromStocks(buffers, MOCK_STOCKS, 8, 55, width, height)` where width/height come from parent container
     - minRadius=8 (smallest bubble ~16px diameter), maxRadius=55 (largest ~110px diameter) -- these produce reasonable sizes for 413 stocks on a typical screen. Adjust if needed during visual tuning.
   - Create physics state: `createPhysicsState(MOCK_STOCKS.length)`
   - Run initial placement: `initialPlacement(buffers, count, width, height)`
   - Store all refs (buffers, physicsState, count) in useRef to avoid React re-renders

2. **Game loop (useEffect on mount):**
   - Create game loop with:
     - `update(dt, time)`: calls `stepPhysics(physicsState, buffers, count, dt, time)`
     - `render()`: calls the render function that draws all bubbles
   - Start the loop, return cleanup that calls `stop()`

3. **Render function (called by game loop each frame):**
   - Get canvas context
   - Apply HiDPI scaling: `setTransform(dpr, 0, 0, dpr, 0, 0)` (same pattern as Phase 1)
   - Clear canvas with `#1a1a1a` background
   - Loop through all bubbles (i = 0 to count):
     - Read `x[i]`, `y[i]`, `radius[i]` from buffers
     - Draw base circle: `ctx.beginPath(); ctx.arc(x, y, r, 0, PI*2); ctx.fillStyle = 'hsl(210, 60%, 35%)'; ctx.fill();`
     - Draw 3D gradient overlay (same radial gradient pattern from Phase 1 drawTestBubble):
       ```
       createRadialGradient(cx - r*0.18, cy - r*0.22, r*0.05, cx, cy, r)
       stops: 0->rgba(255,255,255,0.14), 0.3->rgba(255,255,255,0.05), 0.6->rgba(255,255,255,0), 1->rgba(0,0,0,0.22)
       ```
     - Skip ticker text for now (Phase 3 adds text rendering)
   - Do NOT use ctx.save()/ctx.restore() in the per-bubble loop (performance -- no clipping needed without text)
   - Do NOT use ctx.shadowBlur in the per-bubble loop (expensive per-bubble -- glow is Phase 3 concern)

4. **Resize handling (ResizeObserver):**
   - Keep the ResizeObserver on parent container from Phase 1
   - On resize: update canvas dimensions (width/height + dpr scaling), then call `handleResize(physicsState, buffers, count, newWidth, newHeight)` to clamp bubbles and update spatial hash dimensions
   - Do NOT restart the game loop on resize -- it runs continuously

5. **Component structure:**
   ```tsx
   export function BubbleCanvas() {
     const canvasRef = useRef<HTMLCanvasElement>(null);
     const stateRef = useRef<{ buffers, physics, count, loop } | null>(null);
     // ... setup in useEffect
     return (
       <div className="relative flex-1 overflow-hidden">
         <canvas ref={canvasRef} className="block" />
       </div>
     );
   }
   ```
   - Same JSX structure as Phase 1 (flex-1 overflow-hidden wrapper)
   - All simulation state in refs, not React state (no re-renders during 60fps loop)

CRITICAL constraints:
- No React state for simulation data (buffers, positions, velocities)
- Game loop cleanup on unmount (cancelAnimationFrame)
- HiDPI scaling via setTransform (not ctx.scale) -- matches Phase 1 pattern
- The render function runs at display refresh rate, physics at fixed FIXED_DT -- they are decoupled (PHYS-04)
