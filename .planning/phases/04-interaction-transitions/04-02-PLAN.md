**Part A: Export `fetchStockHistory` from `stockApi.ts`**

The existing `fetchVpsHistory` is private. Create and export a public wrapper:

```typescript
export async function fetchStockHistory(
  ticker: string
): Promise<{ t: number[]; c: number[]; h: number[]; l: number[]; v: number[] } | null> {
  const today = new Date();
  const yearAgo = new Date(today);
  yearAgo.setFullYear(yearAgo.getFullYear() - 1);
  return fetchVpsHistory(ticker, toUnix(yearAgo), toUnix(today));
}
```

This returns 1 year of daily price bars. Returns null if API fails or no data.

**Part B: Create `DetailChart.tsx` -- Canvas-drawn area chart**

A React component that renders an area chart using a `<canvas>` element. No charting library -- use Canvas 2D API directly (project is already canvas-based, zero extra dependencies).

Props:
```typescript
interface DetailChartProps {
  ticker: string;  // Triggers data fetch when changed
}
```

Implementation:
1. On mount / when ticker changes, call `fetchStockHistory(ticker)` to lazy-load price data
2. Show a simple "Loading..." text or subtle spinner while fetching
3. When data arrives, render on canvas:
   - Area chart: line + gradient fill below (green if price went up overall, red if down)
   - Y-axis: min and max price labels
   - High/Low annotations: small dot + label at the highest and lowest price points
   - Chart fills available width, ~180px height
4. HiDPI canvas scaling (same setTransform pattern as BubbleCanvas)
5. If fetch fails, show "Chart unavailable" text
6. Dark theme: dark background matching panel, light gridlines

Drawing the area chart:
```typescript
// Draw filled area under line
ctx.beginPath();
ctx.moveTo(x, y + h); // bottom-left
for (let i = 0; i < closes.length; i++) {
  const px = x + (i / (closes.length - 1)) * w;
  const py = y + h - ((closes[i] - min) / range) * h;
  ctx.lineTo(px, py);
}
ctx.lineTo(x + w, y + h); // bottom-right
ctx.closePath();
// Gradient fill
const grad = ctx.createLinearGradient(x, y, x, y + h);
const color = overallUp ? '#22ec6c' : '#ff4336';
grad.addColorStop(0, color + '40');
grad.addColorStop(1, color + '08');
ctx.fillStyle = grad;
ctx.fill();
// Draw line on top
ctx.beginPath();
for (let i = 0; i < closes.length; i++) {
  const px = x + (i / (closes.length - 1)) * w;
  const py = y + h - ((closes[i] - min) / range) * h;
  i === 0 ? ctx.moveTo(px, py) : ctx.lineTo(px, py);
}
ctx.strokeStyle = color;
ctx.lineWidth = 1.5;
ctx.stroke();
```

Add High/Low markers: find max/min indices, draw small circles with labels (e.g., "H: 85.2" and "L: 62.1") positioned above/below the respective points.

**Part C: Create `DetailPanel.tsx` -- Stock detail overlay**

A React DOM component that renders when `selectedStock` is non-null from `useAppStore`.

Structure (per user decisions from CONTEXT.md):
```
<fixed overlay (full screen, z-50)>
  <backdrop (bg-black/50, backdrop-blur-sm, pointer-events-none)>
  <panel container (centered, max-w-[400px], slide-down animation)>
    <header row>
      <stock logo (from finance.vietstock.vn/image/{TICKER})>
      <ticker + company name>
      <exchange badge (HOSE/HNX/UPCOM)>
      <close X button>
    </header>
    <price display (VND format, e.g., "75,500d")>
    <stats row (Market Cap, Volume)>
    <area chart (DetailChart component)>
    <timeframe tabs (Day/Week/Month/Year with % change for each)>
    <vietstock link>
  </panel>
</fixed overlay>
```

Key behaviors per CONTEXT.md locked decisions:
1. **Slide down from top**: CSS @keyframes animation. Define in main.css:
   ```css
   @keyframes slide-down {
     from { transform: translateY(-100%); opacity: 0; }
     to { transform: translateY(0); opacity: 1; }
   }
   ```
   Apply via Tailwind v4 arbitrary animation class or inline style: `animation: slide-down 0.3s cubic-bezier(0.32, 0.72, 0, 1) forwards`.

2. **Background dims + blurs**: `<div className="fixed inset-0 z-50">` with a backdrop child using `bg-black/50 backdrop-blur-sm`. The backdrop must be `pointer-events-none` so tapping through it hits the canvas (allowing tap-on-another-bubble to switch stocks). The panel itself uses `pointer-events-auto`.

3. **Close behavior**:
   - X button calls `setSelectedStock(null)`
   - NO close on tap-outside (per user decision -- could conflict with tap-on-other-bubble)
   - Swipe down gesture: track touch start Y on the panel container. On touchmove, translate panel down by delta Y. On touchend, if delta > 100px, close panel (setSelectedStock(null)). If delta < 100px, snap back with CSS transition.

4. **When panel is open, tapping another bubble switches**: This is handled by Plan 01 -- the pointerup handler on canvas always calls `setSelectedStock(newStock)` on tap, which changes the Zustand state. The panel reactively updates because it reads `selectedStock` from the store.

5. **Desktop: ~400px centered, Mobile: full-width**: `className="mx-auto w-full max-w-[400px] sm:mt-8"`. On mobile (<640px), panel takes full width. Panel should have `max-h-[85vh] overflow-y-auto` to handle small screens.

6. **Dark theme**: `bg-[#1e1e1e]` or `bg-[#252525]` matching app dark aesthetic. Text white/white-60. Border subtle (`border border-white/10`). Rounded corners (`rounded-xl` on desktop, `rounded-b-xl` on mobile). Shadow (`shadow-2xl`).

Content details:
- **Logo**: `<img src="https://finance.vietstock.vn/image/${stock.ticker}" />` with onerror fallback (hide if broken). Size ~40x40px.
- **Price**: Use `Intl.NumberFormat('vi-VN')` to format `stock.price * 1000` (stored in thousands VND) followed by "d". E.g., "75,500d".
- **Market Cap**: Format as billions with 1 decimal: `${stock.marketCap.toLocaleString('vi-VN')}B VND` or similar compact notation.
- **Volume**: Show as "N/A" for now -- current StockData doesn't include volume. (Can be added from VPS API data in future.)
- **Exchange badge**: Small pill/badge showing "HOSE", "HNX", or "UPCOM" with exchange-appropriate color.
- **52-week High/Low**: Compute from chart data when available (max/min of fetched historical closes). Show as "H: XX.X / L: XX.X" with a simple range indicator.
- **P/E, EPS**: Show "N/A" -- not available in current data model. Placeholder for future API.
- **Timeframe tabs**: 4 buttons showing Day/Week/Month/Year. Each displays the % change value for that timeframe (`stock.changeDay`, etc.) with color coding (green positive, red negative). All 4 always visible (not selectable -- they're informational, not toggles).
- **Vietstock link**: `<a href="https://finance.vietstock.vn/{TICKER}-{exchange}" target="_blank" rel="noopener">` styled as a subtle text link.

**Part D: Swipe-down dismissal gesture**

On the panel container element:
- `onTouchStart`: Record `startY = e.touches[0].clientY`, set `swiping = true`
- `onTouchMove`: Calculate `deltaY = e.touches[0].clientY - startY`. If deltaY > 0 (moving down), set panel transform: `translateY(${deltaY}px)` with reduced opacity (`1 - deltaY / 400`). If scrollTop > 0 (panel content scrolled), don't activate swipe.
- `onTouchEnd`: If deltaY > 100, call `setSelectedStock(null)` (dismiss). Otherwise, animate back to `translateY(0)` via CSS transition.

Use `useRef` for swipe state (no re-renders during gesture). Apply transform via ref to DOM element directly for smooth 60fps gesture response.

**Part E: Update `App.tsx`**

Import and render `DetailPanel` as a sibling to the main content area:

```tsx
import { DetailPanel } from './DetailPanel';

export function App() {
  const loadRealData = useStockStore(s => s.loadRealData);
  const loading = useStockStore(s => s.loading);
  const isRealData = useStockStore(s => s.isRealData);

  useEffect(() => { loadRealData(); }, [loadRealData]);

  return (
    <div className="flex h-screen flex-col">
      <Header />
      {loading && !isRealData ? <LoadingScreen /> : <BubbleCanvas />}
      <DetailPanel />
    </div>
  );
}
```

`DetailPanel` renders as `fixed` overlay, so its DOM position doesn't matter for layout -- it floats above everything.
