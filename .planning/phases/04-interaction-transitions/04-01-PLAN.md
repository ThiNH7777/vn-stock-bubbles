**Part B: Add pointer interaction to `BubbleCanvas.tsx`**

Add hit-testing, drag, hover glow, tap detection, cursor styling, and color lerp to the existing BubbleCanvas component. All pointer state lives in plain variables inside the useEffect closure (NOT React state -- no re-renders during 60fps).

**1. Hit-test function** (inside useEffect closure):
```typescript
function hitTest(px: number, py: number): number {
  // Reverse order = topmost bubble (last drawn) has priority
  for (let i = count - 1; i >= 0; i--) {
    const dx = px - buffers.x[i]!;
    const dy = py - buffers.y[i]!;
    const r = buffers.radius[i]!;
    if (dx * dx + dy * dy <= r * r) return i;
  }
  return -1;
}
```

**2. Pointer state variables** (plain vars, not React state):
```typescript
let hoveredIndex = -1;  // Which bubble the pointer is over
let dragState: {
  bubbleIndex: number;
  startTime: number;
  startX: number;
  startY: number;
  isDragging: boolean;
  lastX: number;
  lastY: number;
  lastTime: number;
} | null = null;
```

**3. Pointer event handlers on canvas element:**

Use Pointer Events API (not mouse/touch separately). Coordinates: `px = e.clientX - rect.left`, `py = e.clientY - rect.top`. Do NOT multiply by DPR -- simulation buffers store CSS-pixel positions.

**pointerdown:** Hit-test at pointer position. If hit, call `canvas.setPointerCapture(e.pointerId)` and initialize dragState with `bubbleIndex`, `startTime = performance.now()`, `startX/Y = px/py`, `isDragging = false`, `lastX/Y = px/py`, `lastTime = performance.now()`.

**pointermove:** If no dragState, do hover hit-test and update `hoveredIndex`. Update cursor: `canvas.style.cursor = hoveredIndex >= 0 ? 'pointer' : 'default'`.

If dragState exists: compute displacement from start. If displacement > 5px and not yet dragging, set `isDragging = true` and change cursor to `'grabbing'`. If dragging: move bubble directly to pointer position (`buffers.x[idx] = px; buffers.y[idx] = py`), zero velocity (`buffers.vx[idx] = 0; buffers.vy[idx] = 0`), update lastX/Y/lastTime for momentum calculation.

**pointerup:** If dragState exists:
- If NOT dragging AND elapsed < 200ms: it's a TAP. Call `useAppStore.getState().setSelectedStock(stocks[dragState.bubbleIndex]!)`. Set cursor to `'pointer'`.
- If dragging: RELEASE with momentum. Calculate throw velocity from last position/time delta: `throwVx = (px - lastX) / dt * 16`, scale by 0.3 damping factor, inject into `buffers.vx[idx]` and `buffers.vy[idx]`. Set cursor to `'pointer'`.
- Clear dragState = null.

**pointerleave:** Reset `hoveredIndex = -1`, reset cursor to `'default'`.

**4. Register events** -- add handlers to canvas element. In cleanup function, remove all handlers and release pointer capture.

IMPORTANT: Use `{ passive: false }` for pointer events to prevent default browser drag behavior. Call `e.preventDefault()` in pointerdown handler.

**5. Drag exclusion from physics** -- While dragging, the dragged bubble must be excluded from physics forces. In the game loop update callback, check if dragState is active. If so, after stepPhysics runs, re-set the dragged bubble's position to the current pointer position and zero its velocity. This prevents physics from fighting the drag. (Alternative: modify stepPhysics to skip a pinned index, but post-correction is simpler and avoids changing the physics module.)

**6. Hover glow modification in render function:**

In the existing render loop, modify the glow calculation for the hovered bubble. The existing code calculates `glowT` based on % change magnitude. For the hovered bubble, boost glowT and breathe:

```typescript
const isHovered = (i === hoveredIndex);
const baseGlowT = abs > 1 ? Math.min((abs - 1) / 11, 1) : 0;
const glowT = isHovered ? Math.max(baseGlowT, 0.8) : baseGlowT;
const breathe = isHovered
  ? 1 - 0.25 + 0.25 * Math.sin(animTime * 3.5 + i * 0.3)   // Faster, stronger pulse
  : 1 - 0.15 + 0.15 * Math.sin(animTime * 1.9 + i * 0.3);
```

Also increase the outer glow `spread` by 50% for hovered bubble: `const spread = isHovered ? 48 : 32;`

**7. Color lerp for timeframe transitions:**

Currently, when the timeframe changes, bubble radii animate smoothly (LERP_SPEED = 0.07) but colors snap instantly because `getBubbleColor` reads the new change value immediately.

Add color lerp: Store per-bubble "current display change" values in a Float32Array alongside existing buffers:
```typescript
const displayChange = new Float32Array(count);
// Initialize to current timeframe values
for (let i = 0; i < count; i++) {
  displayChange[i] = getChange(stocks[i]!, lastTimeframe);
}
```

In the `animateRadii` function (or a new `animateColors` function called alongside it), lerp displayChange toward the target change value:
```typescript
function animateColors() {
  const tf = useAppStore.getState().selectedTimeframe;
  for (let i = 0; i < count; i++) {
    const target = getChange(stocks[i]!, tf);
    const cur = displayChange[i]!;
    if (Math.abs(cur - target) > 0.01) {
      displayChange[i] = cur + (target - cur) * LERP_SPEED;
    } else {
      displayChange[i] = target;
    }
  }
}
```

Then in the render function, use `displayChange[i]` instead of `getChange(stock, selectedTimeframe)` for both color calculation and % change text display. This makes colors AND displayed text values animate smoothly.
